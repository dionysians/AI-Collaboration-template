# 探索验证 (/spike)

技术可行性探索，快速验证想法，代码可丢弃。

## 使用场景
- 不确定某个技术方案是否可行
- 需要评估第三方库或 API
- 在 /plan 前需要技术验证
- 原型验证

## 执行流程

1. **明确目标**
   - 探索什么？
   - 成功标准是什么？
   - 时间限制是多少？（建议 30-60 分钟）

2. **自由探索**
   - 不需要写测试
   - 不需要遵循代码规范
   - 不需要错误处理
   - 可以用最快的方式验证

3. **输出可行性报告**

```
## Spike 报告: [主题]

### 目标
[探索的问题]

### 结论
[可行 / 不可行 / 需要更多探索]

### 发现
- [关键发现 1]
- [关键发现 2]

### 建议方案
[如果可行，建议的实现方向]

### 风险
[已知的风险和限制]

### 参考
[相关文档、代码示例、库链接]
```

## 重要规则

- **Spike 代码不合入主分支** — 它是探索，不是实现
- **不写测试** — 这是唯一不需要测试的模式
- **有时间限制** — 到时间就停下来做判断
- **探索完毕后用 TDD 重新实现** — 从测试开始，不要"参考" spike 代码
