# Bug 修复 (/bugfix)

系统化的 Bug 修复流程，强制根因分析。

## 使用方式

```
/bugfix [bug 描述或 issue 链接]
```

## 执行流程

### 1. 复现
- 确认 bug 可稳定复现
- 记录复现步骤
- 如果无法复现，收集更多信息

### 2. 系统调试
- 触发 `systematic-debugging` skill
- 四阶段：根因调查 → 模式分析 → 假设验证 → 实施修复
- **不猜测，不随机修复**

### 3. 回归测试
- 写失败测试证明 bug 存在（RED）
- 修复 bug 让测试通过（GREEN）
- 确保其他测试不受影响

### 4. 验证
- 执行 `/verify` 确认修复
- 检查是否引入新问题

### 5. 提交
- Commit message 格式：`fix: [描述]`
- 关联 issue（如有）

## Quick 模式

Bug 修复使用 Quick 规划模式：
- 不创建 Epic/Story 结构
- 直接进入调试 → 修复 → 验证流程
- 但仍然强制系统化调试（不跳过根因分析）
